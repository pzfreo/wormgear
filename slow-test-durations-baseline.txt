# Slow test suite — baseline durations
#
# Captured 2026-05-23 on the perf/speed-up-slow-tests branch, before
# enabling pytest-xdist parallelisation. Scope: the CI slow-tests job
# (test_layering + test_golden_volumes + test_geometry_determinism).
#
# Wall-clock totals:
#   serial (baseline):  446 s   (7 min 25 s)
#   xdist `-n auto`:    308 s   (5 min 08 s)   ~31% faster
#
# CI uses `-n auto` (#206).
#
# Long pole:
#   `large_za_multistart` golden at 169 s. This is the wall-clock floor
#   — even with infinite parallelism, the suite can't finish faster than
#   the single longest test. Shrinking it would mean changing the
#   golden's design parameters, which would break the equivalence chain
#   pinned at 1e-6 across the 0.0.x → 0.1.0 refactor. Deliberately left
#   intact.
